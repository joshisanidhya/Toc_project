Theory of Computation can feel overwhelming 🌀 — abstract definitions, complex state machines ⚙️ and formal proofs 📝 that seem distant from practice.

So we built a hands-on lab ⚡ where you can actually run your automata, test your grammars, and see computation happen in real time 🕐

Modules

Six units. One simulator.

Everything you need for your TOC course, from finite automata to the limits of computation.

DFA Simulator

Build deterministic finite automata with custom states and transitions. Enter inputs and trace acceptance step by step.

Regular Expressions

Validate strings against regex patterns instantly. Understand which strings belong to a regular language.

Context-Free Grammars

Define production rules and check if strings can be derived. A quick way to verify your CFG constructions.

Pushdown Automata

Simulate stack-based computation. Test balanced parentheses and explore the power of context-free languages.

Turing Machines

The most powerful computation model. Run binary flip operations on an infinite tape and observe the output.

Decidability

Explore the Halting Problem — the foundational proof that some problems can never be solved by any algorithm.

About This Project

This interactive TOC simulator was created as a hands-on learning tool for students at the Indian Institute of Information Technology, Sri City (IIITS).

It covers all six major units of the Theory of Computation course — from finite automata to decidability — letting you move beyond textbook definitions and actually run the computational models you study in class.

Designed and developed by Sanidhya Joshi.

Halting Problem Demo

The Halting Problem proves that no general algorithm can decide whether an arbitrary program will finish running or loop forever. This is one of the foundational results in computability theory — proved by Alan Turing in 1936.
