Theory Recap

Complete definitions, formal notation, closure properties, and worked examples for every TOC module — DFA, Regex, CFG, PDA, Turing Machine, and Decidability.

What is a DFA? Unit 1

A Deterministic Finite Automaton (DFA) is the simplest machine model in the Chomsky hierarchy. It reads an input string symbol by symbol and decides whether to accept or reject it by following exactly one transition from every state for every symbol.

Formal Definition — 5-Tuple

M = (Q, Σ, δ, q₀, F) Q — finite set of states
Σ — finite input alphabet
δ — transition function: Q × Σ → Q
q₀ — start state (q₀ ∈ Q)
F — set of accept states (F ⊆ Q)

Key Properties

Deterministic Total function δ Reads each symbol once No stack / memory No ε-transitions

For every state and every symbol there is exactly one transition — no guessing, no backtracking.

Worked Example — L = {w | w ends with 'ab'}

States & Alphabet Q = {q0, q1, q2} Σ = {a, b} q₀ = q0 F = {q2}
Transition Table δ
Stateab
→ q0q1q0
q1q1q2
* q2q1q0
Trace for input "aab" q0 —a→ q1 —a→ q1 —b→ q2 ✓ Accepted
Trace for input "aba" q0 —a→ q1 —b→ q2 —a→ q1 ✗ Rejected (q1 ∉ F)

Closure Properties

Regular languages (recognized by DFAs) are closed under:

  • Union: L₁ ∪ L₂
  • Intersection: L₁ ∩ L₂
  • Complement: L̄
  • Concatenation: L₁ · L₂
  • Kleene Star: L*

DFA vs NFA

  • NFA allows multiple transitions per symbol + ε-moves
  • Every NFA can be converted to an equivalent DFA (subset construction)
  • DFA states ≤ 2ⁿ where n = number of NFA states
  • Both recognize exactly the Regular Languages

Pumping Lemma for Regular Languages

If L is regular, ∃ pumping length p such that every string s ∈ L with |s| ≥ p can be split into s = xyz where:

1. |y| ≥ 1 2. |xy| ≤ p 3. ∀ i ≥ 0: xyⁱz ∈ L

Used to prove a language is NOT regular by contradiction.

What are Regular Expressions? Unit 2

A Regular Expression (RE) is an algebraic notation for describing regular languages. Every RE denotes exactly the same class of languages as DFAs and NFAs.

Primitive Operations

ε — the empty string
∅ — the empty language
a — literal symbol a ∈ Σ
R₁ | R₂ — Union (R₁ OR R₂)
R₁ · R₂ — Concatenation
R* — Kleene star (0 or more)

Shorthand / Extended Notation

R⁺ ≡ R·R* (one or more)
R? ≡ R|ε (zero or one)
[abc] ≡ a|b|c (character class)
[a-z] ≡ any lowercase letter
. ≡ any single character
^R ≡ anchored at start
R$ ≡ anchored at end

Precedence (high → low)

1. Kleene star * 2. Concatenation · 3. Union |

Use parentheses to override: (a|b)*abb — strings over {a,b} ending in abb.

Worked Examples

L = strings starting with 'a' a(a|b)*
L = strings with even number of a's (b*ab*ab*)*
L = strings not containing 'bb' (a|ba)*(b|ε)
L = binary multiples of 3 (0|(1(01*0)*1))*

Arden's Lemma

Used to derive a regular expression from a system of equations. If P does not contain ε, then the equation X = QP* ∪ R has unique solution X = QP* ∪ R. Used in constructing RE from DFA state equations.

What is a CFG? Unit 3

A Context-Free Grammar (CFG) generates context-free languages (CFL) — a strictly larger class than regular languages. CFGs are used to describe the syntax of programming languages.

Formal Definition — 4-Tuple

G = (V, Σ, R, S) V — finite set of variables (non-terminals)
Σ — terminal alphabet (V ∩ Σ = ∅)
R — finite set of production rules
— each rule: A → α, A ∈ V, α ∈ (V∪Σ)*
S — start variable (S ∈ V)

Derivation

A string w is in L(G) if there is a sequence of rule applications from S to w:

S ⇒ α₁ ⇒ α₂ ⇒ ... ⇒ w
  • Leftmost derivation: always expand leftmost variable
  • Rightmost derivation: always expand rightmost variable

Worked Example — L = {aⁿbⁿ | n ≥ 0}

Grammar rules S → aSb | ε
Derivation for "aabb" S ⇒ aSb ⇒ aaSbb ⇒ aabb ✓

Note: this language is not regular — no DFA can recognize it. But a CFG can!

Chomsky Normal Form (CNF)

Every CFG can be converted to CNF where every rule is:

A → BC (two variables)
A → a (one terminal)
S → ε (only if ε ∈ L)

CNF is used by the CYK parsing algorithm (O(n³)).

Closure Properties of CFLs

Union Concatenation Kleene Star Intersection Complement

The intersection of two CFLs is not always a CFL. The intersection of a CFL with a regular language IS always a CFL.

Ambiguity

A grammar is ambiguous if some string has more than one leftmost derivation tree (parse tree). Some CFLs are inherently ambiguous — no unambiguous grammar exists for them.

Classic ambiguous grammar E → E + E | E * E | (E) | id

What is a PDA? Unit 4

A Pushdown Automaton (PDA) is a finite automaton augmented with an infinite stack. The stack gives it enough memory to recognize context-free languages — exactly the languages generated by CFGs.

Formal Definition — 6-Tuple

P = (Q, Σ, Γ, δ, q₀, F) Q — finite set of states
Σ — input alphabet
Γ — stack alphabet
δ — transition function:
Q × Σε × Γε → P(Q × Γε)
q₀ — start state
F — accept states

Acceptance Modes

  • Accept by final state: reach an accept state after reading entire input
  • Accept by empty stack: stack is empty after reading entire input

Both modes are equivalent — any language accepted by one mode is accepted by the other.

Worked Example — L = {aⁿbⁿ | n ≥ 1}

Informal description State q0: Push 'A' onto stack for each 'a' read State q1: Pop 'A' from stack for each 'b' read Accept when stack is empty and input is ε
Trace for "aabb" q0, a, Z₀ → q0, AZ₀ (push A) q0, a, A → q0, AA (push A) q0, b, A → q1, ε (pop A) q1, b, A → q1, ε (pop A) q1, ε, Z₀ → qf, Z₀ ✓ Accepted

PDA vs DFA

  • PDA has an unbounded stack — DFA has no memory
  • PDA can count (using stack depth)
  • PDAs are non-deterministic in general
  • Deterministic PDAs (DPDAs) recognize a strict subset of CFLs

CFG ↔ PDA Equivalence

Every CFL has a PDA. Every PDA has a CFG. The conversion algorithms are:

  • CFG → PDA: Use a simulating PDA that mimics leftmost derivation
  • PDA → CFG: Construct variables [p,A,q] for each stack symbol A popped from state p to q

What is a Turing Machine? Unit 5

A Turing Machine (TM) is the most powerful computational model. It has an infinite read/write tape and a finite control. A TM can simulate any algorithm — it defines the class of recursively enumerable (RE) languages.

Formal Definition — 7-Tuple

M = (Q, Σ, Γ, δ, q₀, q_acc, q_rej) Q — finite set of states
Σ — input alphabet (⊔ ∉ Σ)
Γ — tape alphabet (Σ ∪ {⊔} ⊆ Γ)
δ — Q × Γ → Q × Γ × {L,R}
q₀ — start state
q_acc — accept state
q_rej — reject state (q_acc ≠ q_rej)

How it works

  • Head starts at leftmost symbol of input
  • At each step: read symbol, write symbol, move L or R
  • Accept: enter q_acc at any point
  • Reject: enter q_rej, or loop forever
  • Three outcomes: Accept, Reject, Loop

Worked Example — Binary Flip (0↔1)

Transition rules δ(q0, 0) = (q0, 1, R) — flip 0→1, move right δ(q0, 1) = (q0, 0, R) — flip 1→0, move right δ(q0, ⊔) = (q_acc, ⊔, R) — blank = end of input, accept
Trace for input "0110" Tape: [0][1][1][0][⊔] Step 1: q0 reads 0 → write 1, move R → [1][1][1][0][⊔] Step 2: q0 reads 1 → write 0, move R → [1][0][1][0][⊔] Step 3: q0 reads 1 → write 0, move R → [1][0][0][0][⊔] Step 4: q0 reads 0 → write 1, move R → [1][0][0][1][⊔] Step 5: q0 reads ⊔ → q_acc ✓ Output: "1001"

Language Classes

  • Recursive (Decidable): TM always halts (accept or reject)
  • RE (Recognizable): TM may loop; accepts all strings in L but may not reject strings ∉ L
  • Decidable ⊂ RE ⊂ All Languages

Church–Turing Thesis

Any algorithm that can be described informally can be implemented by a Turing Machine. This is a philosophical claim, not a theorem — it cannot be formally proved but is universally accepted.

Variants (all equivalent in power)

Multi-tape TM Non-deterministic TM Enumerator TM 2-stack PDA RAM model

All these variants can be simulated by a standard single-tape deterministic TM. They differ in efficiency, not in power.

What is Decidability? Unit 6

A language L is decidable (recursive) if there exists a Turing Machine that always halts and correctly accepts strings in L and rejects strings not in L. Decidability asks: "Can this problem be solved by any algorithm?"

Decidable Languages

  • ADFA = {⟨M,w⟩ | M is a DFA that accepts w}
  • ACFG = {⟨G,w⟩ | G is a CFG that generates w}
  • EMPTYDFA = {⟨M⟩ | L(M) = ∅}
  • EQDFA = {⟨M₁,M₂⟩ | L(M₁) = L(M₂)}

Undecidable Languages

  • ATM = {⟨M,w⟩ | M accepts w} — RE but not decidable
  • HALTTM = {⟨M,w⟩ | M halts on w}
  • EQTM = {⟨M₁,M₂⟩ | L(M₁) = L(M₂)}
  • All non-trivial semantic properties of TMs (Rice's Theorem)

The Halting Problem — Proof by Contradiction

Assume a TM H decides HALTTM. Construct TM D:

D on input ⟨M⟩: Run H on ⟨M, ⟨M⟩⟩ If H accepts → D rejects (loop) If H rejects → D accepts

Now run D on ⟨D⟩:

If D accepts ⟨D⟩ → H says D halts on ⟨D⟩ → D should loop — contradiction! If D rejects ⟨D⟩ → H says D loops on ⟨D⟩ → D should accept — contradiction!

Therefore no such H can exist. The Halting Problem is undecidable. ∎

Reductions

A language A reduces to B (A ≤ₘ B) if there is a computable function f such that:

w ∈ A ⟺ f(w) ∈ B

If A ≤ₘ B and B is decidable → A is decidable.
If A ≤ₘ B and A is undecidable → B is undecidable.

Rice's Theorem

Every non-trivial semantic property of Turing machines is undecidable.

  • Semantic: about L(M), not the description of M
  • Non-trivial: true for some TMs, false for others

Examples: "Does M accept ε?", "Is L(M) finite?", "Is L(M) = Σ*?" — all undecidable by Rice's.
