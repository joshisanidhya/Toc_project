P vs NP — Interactive Cards (hover to flip)

CLASS P
Polynomial Time
Problems solvable efficiently by a deterministic TM
P = {L | L is decidable in O(nᵏ) time}

Examples: sorting, shortest path (Dijkstra), matrix multiplication, primality testing (AKS). These are "easy" problems — we can find solutions quickly.
CLASS NP
Nondeterministic Polynomial
Solutions are verifiable in polynomial time
NP = {L | L is decidable by a nondeterministic TM in O(nᵏ)}

Examples: SAT (boolean satisfiability), Hamiltonian path, graph coloring, subset sum. We can CHECK a solution quickly, but finding one may be hard.
NP-COMPLETE
Hardest Problems in NP
Every NP problem reduces to these in poly time
NP-Complete = NP ∩ NP-Hard

If ANY NP-complete problem is in P, then P = NP. First proved: SAT (Cook-Levin, 1971). Others: 3-SAT, vertex cover, TSP-decision, clique problem.
THE BIG QUESTION
Does P = NP?
$1M Millennium Prize Problem — unsolved since 1971
Most experts believe P ≠ NP

If P = NP: cryptography breaks, AI could solve complex problems instantly, proofs could be found automatically.
If P ≠ NP: hard problems stay hard. We don't know which. No proof exists either way.
P ⊆ NP  |  P ⊆ co-NP  |  P = NP ?  |  NP ∩ co-NP = ?

❌ Halting Problem Demo

Walk through Turing's diagonalization proof step by step. Click Reveal Next Step to animate the contradiction.

1
Assume H exists
Suppose a TM H exists that decides HALTTM: for any TM M and input w, H accepts if M halts on w, and rejects if M loops forever.
2
Construct D from H
Build a new TM D that takes a TM description ⟨M⟩ as input, runs H on ⟨M, ⟨M⟩⟩ (M on its own description), then does the OPPOSITE: if H says M halts → D loops; if H says M loops → D halts.
3
Run D on itself: D(⟨D⟩)
What happens when we give D its own description? D runs H on ⟨D, ⟨D⟩⟩. Case 1: H says D halts → D loops. But H said D halts. Contradiction!
4
The Other Case
Case 2: H says D loops → D halts. But H said D loops. Contradiction again! Both cases lead to contradiction.
5
Conclusion ∎
Since both cases are impossible, our assumption that H exists must be false.
∴ HALTTM is undecidable. No algorithm can solve the Halting Problem for all inputs.

NP Problems Gallery

SAT (Boolean Satisfiability)
Given a boolean formula (x₁ ∨ ¬x₂) ∧ (x₂ ∨ x₃), is there an assignment making it TRUE?
NP-complete — Cook-Levin theorem (1971)
Hamiltonian Path
Does a graph have a path visiting every vertex exactly once? Checking YES: easy. Finding one: exponential in worst case.
NP-complete
TSP Decision (Travelling Salesman)
Is there a tour of all cities with total distance ≤ k? Brute force: O(n!) paths.
NP-complete
Subset Sum
Given a set of integers, does any subset sum to exactly target T? Used in cryptography.
NP-complete (weakly)

All these problems become polynomial. RSA encryption breaks. Proteins fold in seconds. AI finds optimal solutions. The world changes entirely.
